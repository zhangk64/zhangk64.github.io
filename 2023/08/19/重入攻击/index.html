重入攻击

  • Solidity

介绍

重入攻击（Reentrancy Attack）是一种常见的智能合约安全漏洞，当一个函数对另一个不受信任的合约进行外部调用时，就可能发生重入攻击。

重入攻击的本质是合约内部调用的函数未能恰当地处理合约状态的更新，攻击者利用这个漏洞，将攻击代码插入到合约执行流程中，使得攻击者可以在合约还未完成之前再次调用某个函数(如: fallback, receive)，从而让攻击者在合约中获得额外的资产或信息。可以认为合约中所有的外部调用都是不安全的，都有可能存在重入漏洞。

重入攻击类似编程里的递归调用，发生重入攻击的条件有2个：

  • 调用外部的合约且该合约是不安全的；
  • 外部合约的函数调用早于状态变量的更新；

著名的重入攻击事件：
2016 年，The DAO 合约被重入攻击，黑客盗走了合约中的 3,600,000 枚 ETH，并导致以太坊分叉为 ETH 链和 ETC（以太经典）链。
2019 年，合成资产平台 Synthetix 遭受重入攻击，被盗 3,700,000 枚 sETH。
2020 年，借贷平台 Lendf.me 遭受重入攻击，被盗 $25,000,000。
2021 年，借贷平台 CREAM FINANCE 遭受重入攻击，被盗 $18,800,000。
2022 年，算法稳定币项目 Fei 遭受重入攻击，被盗 $80,000,000。

漏洞示例

先介绍几个概念

receive函数和fallback函数

Solidity支持两种特殊的回调函数，receive() 和 fallback() 。(在0.6版本之后 fallback() 函数拆分成 receive() 和 fallback() 两个函数)

receive函数，不能有任何的参数，不能返回任何值，不需要function关键字，必须包含external和payable，一个合约最多只有一个receive()函数。

  • 原型：receive() external payable { ... };
  • 功能：接收外部转账时需要实现此函数；

fallback函数，不需要function关键字，必须由external修饰，payable修饰符是可选的，一个合约至多含有一个fallback()函数。

  • 原型：fallback() external [payable] { ... };
  • 功能：调用合约不存在的函数时触发fallback函数执行；

转账函数

  1. transfer

    • 函数原型：<address payable>.transfer(uint256 amount);
    • 如果函数执行异常会转账失败，抛出异常（等价于require(send()）(合约地址转账)，会回滚交易（revert);
    • 有gas限制，最大2300（对方合约的fallback或receive函数不能实现太复杂的逻辑，否则gas不够）;

  2. send

    • 函数原型：<address payable>.send(uint256 amount) returns (bool);
    • 如果函数执行异常会转账失败，仅会返回false，不会终止执行（合约地址转账）;
    • 有gas限制，最大2300（对方合约的fallback或receive函数不能实现太复杂的逻辑，否则gas不够）;

  3. call

    • 函数原型：<address>.call(bytes memory) returns (bool, bytes memory)；
    • 如果函数执行异常会转账失败，金辉返回false，不会终止执行（调用合约的方法并转账）；
    • 没有gas限制（支持对方合约的fallback、receive和其它函数实现复杂的逻辑）；

修复建议

检查-效果-交互（checks-effect-interaction）

  • 先检查状态是否满足条件。以Bank合约为例，即 require(balance > 0, "Insufficient balance");
  • 再更新状态。以Bank合约为例，即 balanceOf[msg.sender] = 0;
  • 最后再和其它合约进行交互。以Bank合约为例，即(bool success, ) = msg.sender.call{value: balance}("");
